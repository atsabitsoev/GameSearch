# ADR-004 — Launch with CS2 and Dota 2 Only

- **Status**: Accepted
- **Date**: 2026-05-25
- **Owner**: Product / iOS team

## Context

PandaScore поддерживает 13 игр: CS2, Dota 2, LoL, Valorant, R6, Overwatch, CoD MW, Rocket League, KingOfGlory, LoL Wild Rift, StarCraft, PUBG, Mobile Legends.

При проектировании MVP возник вопрос — стоит ли поддерживать сразу несколько игр (например, добавить Valorant и LoL для расширения аудитории), или ограничиться двумя.

Заготовка `PandaScoreTournamentsService` уже работала только с CS2 и Dota 2. Это задало неявный baseline.

## Decision

В MVP и Phase 1-2 поддерживаем **только CS2 и Dota 2**.

LoL, Valorant и другие игры — Phase 5+ (Future), после стабилизации MVP и первого фидбека от аудитории.

## Alternatives Considered

### А. Все 13 игр сразу
- **Плюсы**: широкое покрытие, привлекает разные аудитории.
- **Минусы**:
  - Лимит 1000 req/hour распределяется на 13 игр — 77 req/hour на игру. Узко.
  - 13 game-acccent цветов, 13 логотипов, 13 разных правил отображения матчей (карты в CS, hero pick в Dota, champion pick в LoL).
  - UI становится перегруженным сегмент-контролом из 13 вариантов.
  - QA — 13× нагрузки.
  - Disk-кэш растёт в 13 раз быстрее.
- **Вердикт**: scope creep, риск выпустить плохо для всех игр вместо хорошо для двух.

### Б. CS2 + Dota 2 + Valorant
- **Плюсы**: захватываем Valorant-аудиторию, тренд растёт.
- **Минусы**:
  - Valorant-специфика: rounds, agents, sites — нужен отдельный display layer.
  - PandaScore покрытие Valorant в free-tier ограничено по сравнению с CS2.
  - Раздувание MVP scope на 30-40%.
- **Вердикт**: разумно для Phase 5, но не для MVP.

### В. Только CS2
- **Плюсы**: ещё уже фокус, проще тестировать.
- **Минусы**: убираем существующую Dota 2-функциональность из placeholder, теряем уже работающую логику.
- **Вердикт**: шаг назад, не оправдан.

### Г. CS2 + Dota 2 (выбранный)
- **Плюсы**:
  - Две самых популярных игры с самым активным турнирным календарём.
  - Заготовка уже есть.
  - 500 req/hour на игру — комфортный буфер.
  - Похожая структура матчей (BoX с картами/играми) — общий display.
- **Минусы**:
  - LoL и Valorant фанаты не получают сервиса (но они и так пользуются нашим приложением для других целей — клубы и новости).

## Rationale

### Фокус продукта
Лучше **сделать хорошо для двух игр**, чем посредственно для пяти. После MVP и фидбека:
- Если аудитория просит конкретную игру — добавляем её следующей.
- Если CS2/Dota 2 покрывают 80%+ интереса — не торопимся расширяться.

### Технический долг
Каждая игра — это:
- Свои особенности display (карты для CS, picks для Dota, agents для Valorant, champions для LoL).
- Свои PandaScore-эндпоинты (хотя структура общая, есть game-specific нюансы).
- Свои тесты.
- Свои фикстуры.

Двумя играми мы покрываем 70%+ типичного use case и устанавливаем шаблон, по которому легко добавлять следующие.

### Архитектура готова к расширению
`Game` enum, `pandaScorePrefix` маппинг, `accentColor` — всё спроектировано так, что добавление новой игры = новый case + ассеты + тесты, без переписывания UI.

## Consequences

### Положительные
- ✅ Сфокусированный, законченный MVP.
- ✅ Лимит API распределяется на меньшее количество запросов.
- ✅ UI остаётся простым — два сегмента вместо 5-13.
- ✅ Можем углубиться в game-specific UX (например, отображение карт CS2 с иконками map veto).
- ✅ Меньше QA-нагрузки.

### Отрицательные
- ❌ Не привлекаем LoL/Valorant/R6 аудиторию на старте.
- ❌ Конкуренты (Liquipedia) поддерживают 9 игр — против них мы выглядим уже.
- ❌ Когда добавляем новую игру в Phase 5+ — нужна полная регрессия (хотя архитектура готова).

### Митигации
- В onboarding и App Store description явно указываем: «CS2 и Dota 2». Не обманываем ожиданий.
- В Settings будущая фича «Игра по умолчанию» позволит пользователю с одной игрой не видеть лишний сегмент.
- Бэклог в `features-matrix.md` фиксирует future-игры — не теряем как идею.

## Roadmap to expansion

Когда добавляем новую игру:

1. Создать ассеты `<game>_logo` в `Assets.xcassets`.
2. Добавить case в `Game` enum + `pandaScorePrefix`, `displayName`, `accentColor`.
3. Расширить `GameSegmentControl` динамически по `Game.allCases`.
4. Добавить game-specific display logic (например, для LoL — drafts) — если требуется.
5. Тесты mapper'а на новые DTO-фикстуры от PandaScore.
6. Обновить App Store screenshots и description.
7. Обновить ADR-004 со статусом `Superseded` и сослаться на новое решение.
