# 09 — UX Principles

Принципы и правила user experience для модуля «Турниры». Каждое UI-решение должно соответствовать этим правилам, иначе оно противоречит продукту.

---

## 10 правил UX модуля

### 1. Speed wins. Always.
- Любой переход — `< 800 ms` при тёплом кэше, `< 2 s` при cold start.
- Skeleton показывается через `200 ms` (раньше — мерцает, позже — кажется зависшим).
- Список рендерится сразу с placeholder-карточками, реальные данные «втекают» в существующие слоты.
- Stale-while-revalidate: если есть устаревший кэш — показываем его сразу, фоном обновляем.

### 2. Spoiler-safe by default.
- Прошедшие матчи показывают команды и время, но **счёт скрыт под пеленой** (`8 — 13` → `▓ — ▓`).
- Раскрытие — тап на счёт или глобальный тогл в Settings.
- При первом запуске — onboarding-плашка «Мы прячем счёт, чтобы не спойлерить. Можно отключить в настройках».

### 3. Live = заметно, но не агрессивно.
- Пульсирующая точка `●` красного цвета + текст `LIVE` рядом с матчем — единственный визуальный сигнал.
- Никаких звуков, вибраций, push-плашек на главной.
- Анимация пульсации — `1.5s` плавно, не мерцание.

### 4. Один экран — одна задача.
- Список турниров = только список. Не вставляем в середину «А ещё купи премиум».
- Детали турнира = только этот турнир. Не показываем «Похожие турниры» в скролле.
- Детали матча = только этот матч. Не показываем «Другие матчи команды» в Phase 1.

### 5. Минимум модальных окон.
- Никаких alert popup (`UIAlertController`) кроме критических ошибок.
- Фильтры открываются как `sheet` (полупрозрачный), не fullScreenCover.
- Stream-выбор — inline список, не bottom sheet.

### 6. Жесты — нативные.
- Swipe-back для возврата (NavigationStack даёт автоматически).
- Pull-to-refresh — везде, где есть список.
- Long-press на карточке — `ContextMenu` с «Поделиться», «В избранное» (Phase 1).
- Никаких кастомных жестов «свайп влево на карточке для удаления» — это не Mail.

### 7. Темнота — основа.
- Только dark theme. Светлой нет и не будет.
- Любой компонент проверяется в тёмной теме.
- Никаких чёрных бордеров на чёрном фоне — используем `EAColor.secondaryBackground` для разделения.

### 8. Расстояние от пальца до тапа — короче.
- Главные действия (кнопки) — внизу экрана, в зоне большого пальца.
- Карточка-список тапается **вся**, не только текст.
- Минимальный touch target — `44 × 44 pt`.

### 9. Без перегруза информацией.
- На карточке турнира — максимум **5 сущностей** одновременно: лого, название, серия, даты, призовой.
- На карточке матча — максимум **4 сущности**: команды, счёт (или время), tier-бейдж, индикатор live.
- Всё остальное — внутри деталей.

### 10. Эмоция = доверие.
- Лого команд — настоящие, не плейсхолдеры. Если PandaScore не отдал — генерируем gradient-кружок с инициалами.
- Шрифт ника игрока — моноширинный (`SF Mono` или fallback) — чувствуется «геймерским».
- Призовой фонд — крупно, золотой акцент `EAColor.yellow` или `EAColor.orange`.

## Анти-паттерны (что НЕ делать)

| Анти-паттерн | Почему плохо | Что вместо |
|---|---|---|
| Tab bar в детальных экранах | Дублирование навигации, плотность | NavigationStack push |
| Снизу плашка-баннер «Установи нашу другую игру» | Раздражение, отвлечение | Никаких баннеров в MVP |
| Pull-to-refresh с кастомной анимацией ракеты | Перформанс + кринж | Стандартный системный |
| Спиннер посередине экрана на 3 секунды | Кажется, что повисло | Skeleton + stale-while-revalidate |
| Цвета rainbow на турнирной сетке | Хаос | Серая база + один accent на winner |
| Большие лого команд (100×100pt) в списке | Карточка раздувается | 32–40 pt в списке, 80 pt в деталях |
| Все турниры в один список без фильтра | Невозможно ориентироваться | Сегменты + фильтр по игре |
| Push-уведомление «Не забудь нас» когда нет матчей | Анти-retention | Только событийные push |

## Поведение скроллов и листов

- **Списочные экраны**: `ScrollView` или `List` со скрытым background (`EAColor.background`).
- **Pull-to-refresh**: `.refreshable { await viewModel.onPullToRefresh() }`.
- **Infinity scroll**: `onAppear` на последнем элементе → `viewModel.onLoadMore()`.
- **Empty state**: переиспользует `TournamentsPlaceholderView` с разным `microcopy` в зависимости от контекста.
- **Sticky-header**: используется только для Live Strip на главной — горизонтальный скролл live-матчей закреплён сверху.

---

## Поведение навигации

- **TabBar** виден всегда, кроме MatchDetails (тогда скрыт через `.toolbar(.hidden, for: .tabBar)`) — это даёт ощущение «глубокого» экрана.
- **NavigationBar title** — `.inline` всегда. `.large` слишком жирно для нашего стиля.
- **Back-button** — стандартный системный «<» с текстом предыдущего экрана.
- **Deeplink** на турнир/матч из push — open в новом NavigationStack-стеке (не append к существующему).

## Анимации и transitions

| Action | Animation |
|---|---|
| Переход на детали | Default NavigationStack push (slide horizontal) |
| Раскрытие skeleton → реальная карточка | `.transition(.opacity).animation(.easeOut(duration: 0.2))` |
| Появление Live Strip | `.transition(.move(edge: .top).combined(with: .opacity))` |
| Раскрытие spoiler | `.spring(duration: 0.25)` |
| Pulse на LIVE индикаторе | `withAnimation(.easeInOut(duration: 1.5).repeatForever())` |
| Tap-feedback на карточке | `.scaleEffect(isPressed ? 0.97 : 1.0)` + `.animation(.easeOut(duration: 0.1))` |

Никаких эффектов вроде parallax-фона при скролле или 3D-rotation карточек.

## Haptic feedback (Phase 3)

- Тап на карточку → `.light` (`UIImpactFeedbackGenerator(.light)`).
- Pull-to-refresh trigger → `.medium`.
- Добавление в избранное → `.success` (`UINotificationFeedbackGenerator`).
- Удаление из избранного → `.warning`.
- Раскрытие spoiler → нет haptic (тихая операция).

## Accessibility

В MVP — базовый минимум:
- Все интерактивные элементы имеют `accessibilityLabel`.
- Карточки матча — `accessibilityLabel: "FaZe против NaVi, BO5, начало в 15:00"`.
- Live-индикатор — `accessibilityLabel: "Идёт прямо сейчас"`.
- Skeleton — `.accessibilityHidden(true)`.

В Phase 3:
- Dynamic Type до `.xxxLarge` (выше — ломается layout).
- VoiceOver полный flow.
- Reduce Motion — отключение пульсации LIVE.

## Onboarding (Phase 1)

Первый раз когда пользователь открывает таб «Турниры»:
- Краткий sheet (2 экрана):
  - «Следи за матчами CS2 и Dota 2. В реальном времени.»
  - «Подпишись на любимую команду — пришлём уведомление за 15 минут до матча.»
- Кнопка «Понятно» — никаких permission-запросов на этом этапе.
- Push permission запрашивается при первой попытке подписаться на команду.

## Состояния экранов (state matrix)

Каждый экран модуля имеет 4 базовых состояния:

| State | Когда | UI |
|---|---|---|
| `.loading` | Первый запуск, нет кэша | Skeleton |
| `.loaded(data)` | Данные получены | Контент |
| `.empty` | Запрос успешен, 0 элементов | `TournamentsPlaceholderView` с контекстным текстом |
| `.error(uiError)` | Сбой | Empty-view + кнопка «Повторить» |

Никогда не показываем пустой экран без объяснений. Никогда не показываем сырой error.

## Принципы выбора цвета

- **Action / акцент**: `EAColor.purpleAccent` (фиолетовый градиент)
- **CS2-контент**: `EAColor.csColor` (оранжевый акцент)
- **Dota 2-контент**: `EAColor.dotaColor` (красный акцент)
- **Live**: чистый красный (можно через `Color.red` или `EAColor.orange`)
- **Призовой**: `EAColor.yellow`
- **Tier S**: `EAColor.purpleAccent`
- **Tier A**: `EAColor.info2`
- **Tier B/C/D**: `EAColor.textSecondary`
- **Победитель**: `EAColor.yellow` (подсветка)
- **Проигравший**: `EAColor.textSecondary`

Не вводим новые семантические цвета без обсуждения. Если нужен — расширяется `EAColor`, не создаются inline `Color.init(red:green:blue:)`.
