# 10 — Screens

Описание всех экранов модуля с ASCII-wireframes, состояниями и поведением.

> Wireframes — это **схематические эскизы**, а не точная вёрстка. Они показывают композицию, иерархию и расположение. Финальные размеры и отступы — по дизайн-системе (`11-design-system.md`).

│  [Клубы]  [Новости]  ●[Турниры]        │ ← TabBar

```

### Состав
- **GameSegmentControl** (sticky top, под navigation) — два сегмента: CS2 / Dota 2. По умолчанию CS2.
- **Live Strip** (sticky под GameSegment, скрывается при отсутствии live-матчей) — горизонтальный `ScrollView` `LiveMatchChip`.
- **TournamentSegmentControl** — три таба: «Сейчас» (по умолчанию для CS2), «Скоро», «Прошедшие».
- **List of TournamentCard** — вертикальный `ScrollView`, infinity scroll.

### Поведение
- `onAppear` → загрузка running tournaments для CS2 + live matches.
- Pull-to-refresh — сброс кэша L1/L2 для текущего сегмента + игры.
- Тап на game-сегмент — переключение, новый запрос (с кэшем).
- Тап на tournament-сегмент — переключение, новый запрос.
- Тап на TournamentCard → `router.push(.tournamentDetails(id))`.
- Тап на LiveMatchChip → `router.push(.matchDetails(id))`.
- Долгое нажатие на TournamentCard → ContextMenu: «Поделиться», «В избранное» (Phase 1).

### Состояния

| State | UI |
|---|---|
| `.loading` (нет кэша) | Skeleton: 5 пустых карточек + skeleton LiveStrip |
| `.loaded([...])` | Карточки + LiveStrip |
| `.loaded([])` + есть фильтр | Inline `TournamentsPlaceholderView` с текстом «По этой игре сейчас нет [сегмент]» |
| `.error(.noInternet)` | `TournamentsPlaceholderView` + кнопка «Повторить», текст «Нет интернета» |
| `.error(.temporaryIssue)` | То же + кнопка «Повторить», текст «Турниры временно недоступны» |

### Состав
- **TournamentHeaderView** — лого лиги, название серии и стадии, даты, страна, tier-бейдж, призовой.
- **TournamentTabPicker** — четыре вкладки. Использует существующий `SwipeSegmentedView`.

- Pull-to-refresh обновляет данные текущего туронира.
- Тап на матч → push `MatchDetailsView`.
- Тап на команду в «Команды» → push `TeamProfileView` (Phase 2).
- Share-кнопка → `UIActivityViewController` с deeplink `gamesearch://tournament/<slug>`.
- Heart-кнопка (Phase 1) → переключение избранного.

```

### Состав
- **MatchHeaderView** — две команды по бокам, в центре — BoX/Live/счёт.
- **MatchGamesList** — список карт (CS2) или игр (Dota 2) с счётом по каждой.
- **MatchStreamsList** — список доступных стримов с языком и платформой.
- **MatchRostersView** — два блока с ростерами.

### Поведение
- TabBar скрыт на этом экране — даёт ощущение «глубины».
- Тап на StreamRow → попытка открыть native app (Twitch/YouTube), fallback на Safari.
- Тап на игрока → Phase 2 push `PlayerProfileView`.
- Тап на счёт прошедшего матча → раскрытие спойлера (если в режиме spoiler-free).
- Свайп назад — стандартный.

### Поведение для разных статусов матча

| Status | Что показываем |
|---|---|
| `not_started` | Время начала большими буквами вместо счёта. Карты — без статуса. Стримы могут быть пустыми. |
| `running` | Live-индикатор, счёт, текущая карта с пометкой ●LIVE. |
| `finished` | Итоговый счёт. Победитель подсвечен `EAColor.yellow`. Spoiler-free скрывает счёт. |
| `canceled` | Бейдж «Отменён» вместо счёта. Карты не показываем. |
| `postponed` | Бейдж «Перенесён» + новое время если есть. |

## 4. LiveStrip (компонент)

Горизонтальный закреплённый блок с live-матчами на главной.

### Wireframe (детально)

```
 ┌────────┐ ┌────────┐ ┌────────┐ ┌────
 │● LIVE  │ │● LIVE  │ │● LIVE  │ │● LIV
 │        │ │        │ │        │ │
 │ FaZe   │ │ Spirit │ │ OG     │ │ T1
 │  1—0   │ │  ●—●   │ │ 16-14  │ │ ●—●
 │ NaVi   │ │ Cloud9 │ │ T1     │ │ G2
 │        │ │        │ │        │ │
 │ BO5    │ │ BO3    │ │ BO5    │ │ BO3
 └────────┘ └────────┘ └────────┘ └────
   140 pt     140 pt     140 pt    140
```

### Состав
- `LiveMatchChip` — карточка `140 pt` ширины, фиксированная высота `120 pt`.
- На каждой: ●LIVE badge, две команды, счёт по сериям (если есть) или ●—● плейсхолдер, формат.

### Поведение
- `onAppear` родителя — fetch /lives для текущей игры.
- Auto-refresh каждые 30 сек пока экран виден.
- Скрывается полностью если live матчей 0 (а не показывает пустой блок).
- Тап → push MatchDetailsView.

## 8. Empty / Error states

Переиспользуем `TournamentsPlaceholderView` с разным microcopy.

### Empty — нет результатов фильтра
```
        🏆
   По этой игре сейчас
   нет идущих турниров

   [ Посмотреть скоро →  ]
```

### Error — нет интернета
```
        📡
    Нет интернета

  Проверь соединение —
  попробуем ещё раз

   [    Повторить     ]
```

### Error — API недоступен
```
        ⚠️
  Турниры временно
   недоступны

  Это с нашей стороны.
  Уже чиним.

   [    Повторить     ]
```
